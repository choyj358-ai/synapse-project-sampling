## 📚 3개의 문서 소개

### 1️⃣ **조유지_프로젝트_종합_가이드.md** (가장 먼저 읽기)
**목적**: 프로젝트 전체를 이해하기

**포함 내용**:
- 프로젝트 개요 (Synapse가 뭔지)
- 당신의 역할 (Learning Card Owner)
- 기술 스택
- W1~W4 주차별 계획
- API 명세 (어떤 API를 만들어야 하는지)
- 협업 팀 정보 (누구와 협업할지)
- 성공 기준 (언제 끝낸 걸까)

**언제 읽을까?**
- ✅ **프로젝트 시작 전**: 필수!
- ✅ **매주 시작할 때**: 그 주의 Step 확인
- ✅ **막힐 때**: "이게 내 일인가?" 확인

---

### 2️⃣ **CLAUDE_지침_조유지용.md** (Claude와 대화할 때)
**목적**: Claude가 당신의 프로젝트 맥락을 이해하게 하기

**포함 내용**:
- SM-2 알고리즘 수식 (자세히)
- 각 Step의 Definition of Done
- API 명세 (더 기술적으로)
- 개발 팁 (비전공자 맞춤)

**어떻게 사용할까?**
1. **첫 번째 Claude 대화**:
   - "CLAUDE_지침_조유지용.md의 내용을 바탕으로 나를 도와줄 수 있나?"
   - 파일 내용 일부를 복사해서 보여주기

2. **각 Step마다**:
   - 그 Step의 Definition of Done을 Claude에게 보여주기
   - "이게 다 되면 완료인데, 첫 번째 코드 스켈레톤을 짜줄 수 있나?"

3. **막힐 때마다**:
   - 에러 메시지 + 코드 → Claude에게 물어보기
   - 이 문서의 "Definition of Done"과 맞는지 확인

### 3️⃣ **Claude_예시_프롬프트_모음.md** (실제 대화 템플릿)
**목적**: Claude에게 정확하게 물어보는 방법 배우기

**포함 내용**:
- 13개의 실제 상황별 예시 프롬프트
- 각 예시마다 "당신의 질문" + "Claude의 답변" 형식
- Copy-Paste 가능한 코드 템플릿

**예시**:
- "Spring Boot 프로젝트를 어떻게 만들어?"
- "SM-2 알고리즘이 맞는지 검증하고 싶어"
- "Kafka 이벤트가 발행되지 않아 - 디버깅해줄 수 있나?"
- "이 코드 구조가 맞나?" (코드 붙여넣기)

**언제 사용할까?**
- 🔍 "나는 어떻게 Claude에게 물어봐야 할까?" 할 때
- 🎯 "비슷한 상황의 예시가 있을까?" 할 때
- 💾 "이 프롬프트를 복사해서 내 상황에 맞게 수정할까" 할 때

```

   - Java 21 설치
   - Gradle 설치
   - PostgreSQL + Docker 설치
   - GitHub 저장소 클론

3. 💬 Claude 사용 준비
   - CLAUDE_지침_조유지용.md 저장해두기
   - Claude에게 처음 대화할 때 보여주기
```

### Step B: 첫 주 (W1)
```
1. 📋 "조유지_프로젝트_종합_가이드.md"에서 W1 확인
2. ✅ Step 1: 초기 설정
   - 막힐 때마다 Claude_예시_프롬프트_모음.md 예시 1 참고
   - Claude에게 물어보기
3. ✅ Step 2: 덱/카드 CRUD
   - 예시 3, 4 참고
4. ✅ Step 3: SM-2 알고리즘
   - 예시 5, 6 참고
   - "SM-2 알고리즘이 정말 맞나?" 할 때 Claude에게 검증 요청
```

### Step C: 계속 (W2~W4)
```

1. "조유지_프로젝트_종합_가이드.md"에서 그 주 목표 확인

```
