# Claude_Rules

Synapse 프로젝트 개발에서 Claude가 반드시 지켜야 할 규칙입니다.

---

## 필수 프로세스

모든 작업은 다음 순서로 진행:

1. Plan Mode 활성화 선언
2. Socratic 질문 (3개 이상)
3. 구현 계획 제시 (파일 목록 + 코드 미리보기)
4. "이대로 진행할까요?" 물어보기
5. 사용자가 "네" 또는 "승인"이라고 한 후 코드 작성
6. 작업 기록에 자동 기록

---

## 절대 금지

- 사용자 승인 없이 코드 실행 금지
- 설명 없이 코드만 제시 금지
- Socratic 질문 없이 바로 제안 금지
- 계획 없이 바로 코딩 금지
- 완료 후 작업 기록 기록 없이 마무리 금지

---

## 코드 실행 조건

- 사용자가 "네" 또는 "승인"이라고 한 후에만 실행
- 그 전까지 코드 작성 절대 금지
- 파일 생성, 수정, 삭제는 승인 후에만

---

## 특수 상황

**사용자가 "수정 요청"할 때:**
- 계획을 다시 수정해서 제시
- "이대로 진행할까요?" 다시 물어보기
- 승인 받을 때까지 대기

**오류가 발생했을 때:**
- 에러 메시지 분석 후 수정안 제시
- 사용자에게 "이렇게 수정하면 되는데, 진행할까요?" 물어보기
- 승인 후 실행

---

## Synapse 프로젝트 문서 관리

### 작업 시작 전 (필수)

1. docs/rules 숙지
   - docs/rules 내 모든 파일 읽기
   - 프로젝트 기본 규칙 파악

### 작업 완료 후 (필수)

1. HISTORY_learning-card.md 업데이트
   - 표에 상태 수정
   - 시작일, 완료일, 비고 기재
   - 아래 작업로그에 당일 날짜와 내용 기재

2. docs/project-management/workflow/ 확인
   - card 해당 기능 문서 찾기
   - 구현한 항목에 체크 표시

---

## 작업 기록 관리

work-logs/ 폴더(Synapse 프로젝트 내부)에 상세 기록:

파일명 형식: YYYY-MM-DD_W{week}Step{step}_{feature-name}.md

예시:
- 2026-05-14_W1Step1_spring-boot-setup.md
- 2026-05-14_W1Step2_deck-card-crud.md

포함 내용:
- 생성된 파일 목록
- 테스트 결과
- 학습 내용
- 특이사항
